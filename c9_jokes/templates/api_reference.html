Sandbox Terminal API

MethodPathDescription
POST/api/sandbox/exec Execute a shell command in C10, C11, or C12b sandbox.
Body: {"command":"ls -la","sandbox":"c10","timeout":30,"cwd":".","session_id":""}
Returns: {"stdout":"…","stderr":"…","exit_code":0,"timed_out":false}
sandbox: "c10" (agent workspace), "c11" (multi-agent, needs session_id), or "c12b" (lean host-exposed coding/test sandbox)

Container Management API

MethodPathDescription
GET/api/containers List all Docker containers with status and toggleable flag.
Returns: {"ok":true,"containers":[{"name":"…","status":"…","state":"running","toggleable":true,"core":false}]}
POST/api/container/toggle Start or stop an optional container (C2b, C5b, C7ab, C7bb, C8b only — core containers are protected).
Body: {"name":"C2b_agent-terminal","action":"stop"}
Returns: {"ok":true,"name":"…","action":"stop","output":"…"}

Core containers (always protected, cannot be toggled): C1b, C3b, C6b, C9b, C10b, C11b
Optional containers (can be started/stopped): C2b, C5b, C7ab, C7bb, C8b

Task Management Endpoints

MethodPathDescription
GET/api/tasksList all task definitions. Query: ?active=1&template_key=X&limit=100
POST/api/tasksCreate or update a task definition. Body: full task object with id, name, mode, schedule_kind, executor_prompt, workflow_steps, etc.
DELETE/api/tasks/{task_id}Delete a task definition permanently.
POST/api/tasks/{task_id}/runExecute a task immediately (manual trigger).
POST/api/tasks/{task_id}/startStart a scheduled or continuous task.
POST/api/tasks/{task_id}/pausePause an active task run.
POST/api/tasks/{task_id}/resumeResume a paused task run.
POST/api/tasks/{task_id}/stopStop a running task.
POST/api/tasks/{task_id}/completeMark a task run as complete.
POST/api/tasks/{task_id}/repeatRe-run a task with the same parameters.
POST/api/tasks/{task_id}/restartRestart a task from the beginning.
POST/api/tasks/{task_id}/redoRedo the last execution on the same trace.
POST/api/tasks/{task_id}/cloneDuplicate a task definition to a new ID.
POST/api/tasks/{task_id}/archiveSoft-delete (archive) a task definition.
POST/api/tasks/bulkBulk create or update multiple task definitions.
POST/api/tasks/seed-examplesSeed the DB with built-in example tasks.
POST/api/tasks/draft-from-textAI-draft a task definition from plain-English description. Body: {"text":"…"}

Task Run Endpoints

MethodPathDescription
GET/api/task-runsList run history. Query: ?task_id=X&status=running&limit=20
GET/api/task-completedList terminal runs (completed/failed/cancelled). Query: ?task_id=X&status=completed,failed,cancelled&limit=50 — returns enriched items with steps, alerts, feedback, and recovery context.
GET/api/task-pipelinesFull pipeline trace for one or more runs. Query: ?task_id=X&run_id=Y&status=running
GET/api/task-previewPreview output for a task run. Query: ?task_id=X&run_id=Y
POST/api/task-feedbackSubmit agent feedback on a task result. Body: {"task_id":"…","run_id":"…","agent_id":"…","status":"…","summary":"…"}

Task Template Endpoints

MethodPathDescription
GET/api/task-templatesList all task templates. Query: ?source=user&active=1
POST/api/task-templatesCreate or update a task template. Body: template object with key, name, mode, prompts, and execution fields.
POST/api/task-templates/{key}/cloneClone a template into a new task definition.
POST/api/task-templates/{key}/archiveArchive (soft-delete) a task template.

Alerts Endpoints

MethodPathDescription
GET/api/alertsList all alerts with task and run context. Query: ?limit=250 (max 500). Returns LEFT JOIN of task_alerts ↔ task_definitions ↔ task_runs.
POST/api/alerts/{alert_id}/statusUpdate alert status. Body: {"status":"acknowledged|resolved|snoozed|open","snooze_minutes":30}
POST/api/alerts/{alert_id}/ackAcknowledge an alert (shortcut for status=acknowledged).

Alert severities: info, warning, error, critical
Alert statuses: open, acknowledged, resolved, snoozed

Example: Run Agent Task (SSE)

curl -N "http://localhost:6090/api/agent/run?task=write+a+hello+world+flask+app&agent_id=c9-jokes"

Example: Run Shell Command in Sandbox

curl -sS -X POST http://localhost:6090/api/sandbox/exec \
  -H "Content-Type: application/json" \
  -d '{"command":"ls -la","sandbox":"c10","timeout":10}'

Example: Run Shell Command in Lean C12b Sandbox

curl -sS -X POST http://localhost:6090/api/sandbox/exec \
  -H "Content-Type: application/json" \
  -d '{"command":"python3 --version && node --version && uv --version","sandbox":"c12b","timeout":10}'

Example: Stop a Container

curl -sS -X POST http://localhost:6090/api/container/toggle \
  -H "Content-Type: application/json" \
  -d '{"name":"C2b_agent-terminal","action":"stop"}'

Example: Direct C1 Call (streaming)

curl -sS -X POST {{ urls.c1 }}/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "X-Agent-ID: c9-smoke" \
  -d '{"model":"copilot","messages":[{"role":"user","content":"Tell me a joke"}],"stream":true}'

Example: Chat via C9 proxy

curl -sS -X POST http://localhost:6090/api/chat \
  -H "Content-Type: application/json" \
  -d '{"agent_id":"c2-aider","prompt":"Tell me a joke"}'

Example: Chat via C9 proxy (streaming)

curl -N -X POST http://localhost:6090/api/chat \
  -H "Content-Type: application/json" \
  -d '{"agent_id":"c2-aider","prompt":"Tell me a joke","stream":true}'
