## Task 2: Complex Cascading Task with 4 Subtasks (Output Type)

### Current State Analysis
- **Tasked page** (`tasked.html`): Builder for task definitions with workflow steps, templates, LLM-assisted authoring
- **Pipeline page** (`piplinetask.html`): Run-centric monitor showing step results, feedback, events, recovery sessions
- **Alerts page** (`alerts.html`): Alert management with acknowledge/resolve/snooze
- **TaskCompleted page** (`task_completed.html`): Terminal runs with redo/clone/archive actions
- **Preview page** (`tasked_preview.html`): Two-panel sidebar+preview with task/run selection

### DB Tables (All Exist in Single SQLite `c9.db`)
- `task_definitions` — Task definitions
- `task_runs` — Individual task executions
- `task_events` — Lifecycle events
- `task_alerts` — Alert records
- `task_workflow_steps` — DAG workflow steps
- `task_step_results` — Per-step execution results
- `task_feedback_events` — Agent feedback
- `task_templates` — Reusable templates
- `task_run_claims` — Distributed locking
- `session_manager_sessions` — Adaptive timeout/retry tracking
- `session_manager_metrics` — Historical latency metrics

**Note**: All pages share the same `c9.db` SQLite database. No separate DBs are missing. The schema is comprehensive with 20+ tables.

### Cascading Task Design: 4-Subtask Output Task
Create a task named **"Cascading Code Pipeline"** with these 4 sequential subtasks:

| Step | Kind | Description |
|------|------|-------------|
| 1 | Trigger | Recurring every 10 minutes |
| 2 | Sandbox (C12b) | Write a Python Flask app with a `/health` endpoint |
| 3 | Chat (C6 KiloCode) | Review the code, suggest improvements, generate unit tests |
| 4 | Alert | Create alert if tests pass (success) or fail (error) |

### Implementation Steps
1. **Create the task definition** via `POST /api/tasks` with 4 workflow steps
2. **Verify on Tasked page** — Task appears in the table with correct mode/steps
3. **Execute the task** via `POST /api/tasks/{id}/run`
4. **Monitor on Pipeline page** — Watch step-by-step execution
5. **Check Alerts page** — Verify alert creation
6. **Check TaskCompleted page** — Verify terminal run appears
7. **Check Preview page** — Verify task output is viewable
